# Rules for daily-progress.md

- **Append-only.** Never insert items above existing markers or reorder entries.
- **Never add items from GitHub/Linear scans without explicit user confirmation.** Show findings, ask which ones to add and which day they belong to. PR activity date does not mean work was done that day — always ask.
- **Never write reflections.** Present plan vs done analysis, ask the user to write the reflection, use their exact words.
- **Preserve the user's voice.** Never sanitize, professionalize, or remove personal comments or humor.
- **Deduplicate.** If two items refer to the same work or PR, combine into one entry.
